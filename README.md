# CoreStorm Global — Solar & Renewable Energy Website

Single-page marketing site for **CoreStorm Global** (corestormglobal.com), an
Australian trader and exporter of solar panels, inverters, battery storage and
off-grid renewable energy systems — supplying local customers and exporting to
the UK and Europe DDP.

> **Tagline:** *Powering a Greener Future – Australian Solar Solutions for Local and Global Markets*

## Files

| Path | Purpose |
|------|---------|
| `index.html` | Markup — hero, stats, products, export process, projects, about, why-us, testimonials, CTA, contact, footer |
| `css/styles.css` | Design system (green + blue on eco-cream), layout, responsive breakpoints (1024 / 820 / 520px), robust reveal animations |
| `js/main.js` | Sticky header, mobile nav, scroll reveals, animated counters, contact-form validation + simulated submit, image fallback, footer year |
| `assets/img/` | Self-hosted stock photos (generated by `scripts/fetch-assets.js`) |
| `assets/fonts/` | Self-hosted Playfair Display + Inter woff2 and generated `fonts.css` |
| `scripts/fetch-assets.js` | Downloads and resizes all remote imagery/fonts into `assets/` |
| `build.js` | Packages everything into one self-contained, CSP-safe HTML file |
| `build/index.html` | **Generated** self-contained artifact (all images, fonts, CSS, JS inlined) |

## Develop

Open `index.html` directly, or serve the folder:

```bash
python -m http.server 4180     # then visit http://localhost:4180
```

**The site makes zero external requests even in dev** — all images and fonts are
self-hosted under `assets/`, so it renders correctly offline and inside sandboxed
preview panels that block external hosts. If you change the imagery list, re-run:

```bash
node scripts/fetch-assets.js   # re-downloads into assets/ (needs network once)
```

## Build the self-contained artifact

```bash
npm install      # installs sharp (used by fetch-assets)
npm run build    # -> build/index.html   (no network required)
```

The build step:

1. Embeds every `assets/img/*.jpg` the page references as a base64 JPEG data URI.
2. Embeds **Playfair Display** (600 normal + italic, latin subset) as woff2 data
   URIs. Body text falls back to a `system-ui` stack (Inter is not embedded, to
   keep the artifact lean).
3. Inlines `css/styles.css` and `js/main.js`.
4. **Asserts zero remaining external asset URLs** — the output is safe to host under
   a strict Content-Security-Policy that blocks all external requests.

## Robustness notes

- **Reveal animations never blank the page.** Elements are only hidden under
  `html.js .reveal:not(.visible)`; `main.js` adds the `js` class on load, so with
  no JavaScript everything renders visible. The `IntersectionObserver` also reveals
  any element already scrolled above the viewport, so landing mid-page via an anchor
  link or scroll restoration shows all prior content. `prefers-reduced-motion` shows
  everything with no transitions.
- **Image fallback.** If a stock photo fails to load, `main.js` swaps in a themed
  gradient (`img.img-error`) instead of a broken-image icon; the element keeps its
  box so layout never collapses.

## Wiring up the contact form

The form runs a simulated submission. To make it live, either:

- Set the `<form>` `action` to your Formspree URL with `method="POST"`, **or**
- Replace the `setTimeout(...)` stub in `js/main.js` (clearly commented) with a
  `fetch()` to Netlify Forms / your API.

## Imagery caveat

All product/project photos are genuine solar-array / installation photography
(verified against their captions). Free stock has no reliable close-ups of specific
**inverter** or **battery** hardware, so those rows use system/installation imagery.
Drop in supplier product shots for literal inverter/battery close-ups when available —
just replace the relevant `<img src>` values and re-run `npm run build`.
